# AI Video Explainer Generator

A compact, local-first tool to generate short explainer videos from AI images and TTS voiceovers.

This repository provides a FastAPI-based scene editor and a small set of engines to:

- Generate voiceover audio from text (via `edge-tts`).
- Generate images from text prompts using **several selectable backends** (free no-key, free-with-signup, and Google Gemini / Imagen "Nano Banana" family).
- Compose images, audio and transitions into MP4 videos (via `moviepy` or `ffmpeg`).

Features

- Web UI: editing scene cards and rendering from the browser (`/` and `/bulk`).
- **Image generator dropdown** on the Scene Editor and in Bulk step 1 — choice is sent to the API and saved with projects.
- **Project Audio Overlay**: Upload a single audio file that plays continuously across all scenes in your video. Ideal for background music or full-length podcast audio.
- **Audio Part Selection (Trimming)**: Select exactly which part of an uploaded scene audio to use by setting a start and end time. The scene preview playback restricts to your trimmed range.
- API: single-scene and bulk endpoints for image/audio/video generation.
- Two rendering paths: fast FFmpeg-based assembly (bundled automatically, no install needed) and a MoviePy fallback.
- Project save/load and built-in templates.
- **Custom download filenames** — automatically derived from your project name with safe character sanitisation, avoiding blocking browser prompts.
- **Custom project names** — name your project when saving with a confirmation prompt.
- **Project deletion with confirmation** — safely delete projects with a confirmation dialog to prevent accidental loss.
- **Gemini watermark removal** — images generated by any Google Gemini or Imagen provider have their visible diamond/sparkle watermark automatically detected and erased before the video is rendered. A per-scene checkbox also lets you manually flag any uploaded image as Gemini-sourced to trigger the same removal.

### Managing API Keys

The app allows you to use your own API keys without setting environment variables on your server:

1. In the **Scene Editor** (top right) or **Bulk Mode** (header), click the **"API Keys"** button.
2. Enter your keys for the desired services.
3. Click **"Save Keys"**.

- Keys are stored **locally in your browser's localStorage**. They are sent with each generation request but never stored permanently on the server disk.
- If you prefer, you can still use server-side environment variables (e.g., `OPENAI_API_KEY`, `TOGETHER_API_KEY`, `GEMINI_API_KEY`).

Notes:

- **Nano Banana** is Google's name for Gemini's built-in conversational image generation; see the [Image generation](https://ai.google.dev/gemini-api/docs/image-generation) docs. Models and availability can change — if a model ID returns 404, check Google's model list and open an issue or PR to update `image_engine.py`.
- Google image APIs may apply **SynthID** watermarking and regional restrictions (see Google's docs).
- Providers that require a key will **surface errors** in the UI if the key is missing or invalid. Only Pollinations keeps the silent placeholder fallback after retries.

### Gemini watermark removal

Google Gemini and Imagen providers embed a visible diamond/sparkle logo in the bottom-right corner of generated images. The app removes this automatically before encoding the final video.

**How it is triggered:**

- **Automatic** — when you click **Generate** on a scene using any Gemini/Imagen provider, the API response sets a `gemini_source` flag and the per-scene **"Image from Gemini (remove watermark)"** checkbox is checked automatically.
- **Manual** — if you upload an image that was previously generated by Gemini outside the app, tick the **"Image from Gemini (remove watermark)"** checkbox on that scene card before rendering.

**How the removal works:**

1. The bottom-right 15 × 15 % of the image is isolated — the region where the Gemini diamond always appears.
2. Pixels significantly brighter than the local average (the sparkle shape) are detected and their mask is dilated by 4 px to cleanly cover edges.
3. Masked pixels are replaced with the median value of the surrounding unmasked pixels in that region (inpainting).
4. The cleaned image is saved as a `_dewm.jpg` copy alongside the original; the original is never modified.
5. The cleaned copy is what gets encoded into the final video on both the FFmpeg fast path and the MoviePy fallback.

`scipy` is used for the mask dilation step. If it is not installed the app falls back to erasing a fixed 80 × 80 px patch at the very bottom-right corner, which covers the typical watermark position.

## Project Management

### Saving Projects

- Click **"Save"** to save your current project.
- You will be prompted to enter a custom project name.
- Projects are stored in `projects/` directory as JSON files.
- All scene data, media references, and settings are preserved.

### Loading Projects

- Click **"Load"** to open the projects modal.
- View all saved projects and available templates.
- Click **"Load"** on any project to restore it with all scenes and media.
- Click **"Use"** on any template to start from a pre-built template.

### Deleting Projects

- In the projects modal, click the **×** button next to any project.
- A confirmation dialog will appear asking you to confirm deletion.
- Once confirmed, the project JSON and all associated scene cache directories (audio, images, temporary media) are permanently removed to keep disk usage lean.
- This action cannot be undone.

## Video Download & Naming

### Custom Download Filenames

- When rendering completes, the app automatically sanitises your custom project name for the download filename.
- The download button displays the resulting filename (e.g., "Download: my_video.mp4") before you click it.
- Files download directly without jarring, tab-blocking browser prompts.

How it works (high level)

- The FastAPI app (`main.py`) exposes endpoints to generate or upload assets, then assemble them into scenes.
- `audio_engine.py` uses `edge-tts` to synthesize voiceovers to MP3.
- `image_engine.py` routes prompts to the selected provider (`generate_prompt_image(..., provider=...)`). Returns `(path, is_gemini: bool)` so callers know whether watermark removal is needed. Exposes `is_gemini_provider()` helper.
- `video_engine.py` composes scenes with `moviepy`. When all scenes are still images, `ffmpeg_render.py` will attempt a much faster FFmpeg-based path. Contains `_remove_synthid_watermark()` for the MoviePy render path.
- `ffmpeg_render.py` pre-processes images with `_erase_gemini_watermark_inplace()` before encoding when `gemini_source=True` on a scene.
- Generated files are stored under `cache/` (per-scene) and final MP4s are placed in `output/`.

API examples

1. List image providers

```bash
curl -s "http://localhost:8000/api/image-providers"
```

2. Generate a voiceover (async endpoint)

```bash
curl -X POST "http://localhost:8000/api/generate-audio" \
  -F "text=Hello, this is a test voiceover" \
  -F "voice=en-US-JennyNeural"

# Response: {"scene_id":"<id>","audio_url":"/api/media/<id>/voiceover.mp3"}
```

3. Generate an image from a prompt (optional `provider`)

```bash
curl -X POST "http://localhost:8000/api/generate-image" \
  -F "prompt=Golden hour over a mountain lake, cinematic" \
  -F "width=1920" -F "height=1080" \
  -F "provider=gemini_nano_banana_2"

# Response includes gemini_source: true when a Gemini/Imagen provider was used
# {"scene_id":"<id>","image_url":"/api/media/<id>/image.jpg","gemini_source":true}
```

4. Generate an image and render a short motion video from it (single request)

```bash
curl -X POST "http://localhost:8000/api/generate-image-video" \
  -F "prompt=A friendly robot giving a demo" \
  -F "duration=6" \
  -F "animation=ken_burns" \
  -F "provider=stable_horde"

# Response contains a `video_url` such as "/api/media/<scene_id>/video.mp4"
```

5. Upload custom media for a scene (image / video / audio)

```bash
curl -X POST "http://localhost:8000/api/upload-media" \
  -F "file=@/path/to/your/image.jpg" \
  -F "scene_id=optional-scene-id"
```

6. Download a rendered video with custom filename

```bash
curl -X GET "http://localhost:8000/api/download/<job_id>?filename=my_custom_name" \
  -o my_custom_name.mp4
```

Bulk JSON (`/api/bulk-generate` `payload` field) may include `"image_provider": "imagen_fast"` next to `image_source`, `orientation`, etc.

Where files are placed

- `cache/<scene_id>/` — per-scene temporary files (image.jpg, voiceover.mp3, video.mp4, `image_dewm.jpg`, etc.)
- `output/` — final exported MP4s (one-per-rendered-job)
- `projects/` — saved project JSON files (managed by the UI and API)
- `static/` — front-end HTML/CSS/JS (editor UI)

Important implementation notes

- The app prefers FFmpeg when available (`ffmpeg_render.py`). FFmpeg is automatically discovered from the `imageio-ffmpeg` package.
- Image routing is implemented in `image_engine.py` using `aiohttp`. Pollinations is the only backend that falls back to a local placeholder after repeated failures.
- `generate_prompt_image` now returns a `(path, is_gemini: bool)` tuple instead of a plain string. Any custom code calling it must unpack both values.
- Watermark-cleaned images are cached as `<original>_dewm.jpg` in the same scene cache folder. Delete the `_dewm.jpg` file manually if you want the removal to re-run on the same source image.
- TTS uses `edge-tts` which can list available voices with `edge-tts --list-voices`. Audio generation includes a 3-attempt exponential backoff for resilience against transient WebSocket failures.
- The server will create `cache/` and `output/` directories at runtime if they do not exist.
- Download filenames are passed as query parameters to the `/api/download/{job_id}` endpoint and automatically sanitized.
- **Python 3.13 Support:** `video_engine.py` injects a lightweight IPython stub at module load time to bypass MoviePy's deep import chain, avoiding a known `KeyboardInterrupt` crash during startup on Python 3.13.

Security & Privacy

- Uploaded/Generated media is stored locally under `cache/`, `output/` and `static/uploads/`.
- Be careful not to commit large or sensitive media files to Git—see the included `.gitignore` which prevents common media from being added.
- **Do not commit API keys.** Use environment variables or a local `.env` loader in your shell profile.

Troubleshooting

- If you see `FFmpeg not found` or encoding fails: install FFmpeg and ensure it is on your `PATH`.
- If Pollinations often yields placeholders: pick another provider or reduce resolution; you may be rate-limited.
- If Gemini/Imagen returns errors: confirm the model is enabled for your key in Google AI Studio and that billing / quota allows image generation.
- If the Gemini diamond watermark is still visible after rendering: ensure the **"Image from Gemini (remove watermark)"** checkbox is checked on the scene card, then re-render. Also confirm `scipy` and `numpy` are installed (`pip install scipy numpy`).
- If watermark removal leaves a noticeable patch: the watermark position may differ slightly. Delete the `_dewm.jpg` cache file so it is regenerated, or adjust the `rx0`/`ry0` detection thresholds in `_remove_synthid_watermark` (video_engine.py) and `_erase_gemini_watermark_inplace` (ffmpeg_render.py).
- If projects fail to load: ensure all scene media files exist in the `cache/` directory. Deleted cache files will cause load failures.
- If delete confirmation doesn't appear: ensure JavaScript is enabled in your browser and no browser extensions are blocking dialogs.

Extending the project

- Add a provider in `image_engine.py` (`_IMAGE_PROVIDER_META` + branch in `generate_prompt_image`). If the new provider also embeds a visible watermark, add its ID to `_GEMINI_PROVIDER_IDS`.
- Add cloud TTS: replace `audio_engine.generate_audio_from_text` with any other TTS provider that writes MP3 files.
- Add more render presets in `video_engine.RENDER_PRESETS` for quality vs speed tradeoffs.
- Customize download behavior by modifying the `/api/download/{job_id}` endpoint in `main.py`.

Files of interest

- `main.py` — FastAPI app and HTTP endpoints. Includes `/api/download/{job_id}` for custom filename downloads.
- `audio_engine.py` — voice synthesis wrapper (edge-tts).
- `image_engine.py` — prompt → image (multi-provider). Exports `is_gemini_provider()` and returns `(path, is_gemini)` from `generate_prompt_image`.
- `video_engine.py` — scene assembly using MoviePy. Contains `_remove_synthid_watermark()` for the MoviePy render path.
- `ffmpeg_render.py` — fast FFmpeg-based path for still-image scenes. Contains `_erase_gemini_watermark_inplace()` for the FFmpeg render path.
- `project_manager.py` — save/load/list/delete project JSON files in `projects/`.
- `static/app.js` — Scene Editor frontend. Handles project naming, deletion confirmation, and custom download filenames.
- `static/index.html` — Scene Editor HTML. Each scene card has an **"Image from Gemini (remove watermark)"** checkbox in the Visual panel.
